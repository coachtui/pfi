## Completed (previous phase — PWA manifest + Playwright e2e)

- **Two real bugs found and fixed by the new e2e harness on its first run — the headline result of this slice.** (1) `src/proxy.ts`'s auth matcher redirected unauthenticated `/manifest.webmanifest` requests to `/login`, which would have silently broken PWA installability for every real, logged-out visitor (a manifest fetch behind a login wall isn't installable) — fixed by adding `manifest.webmanifest` to the matcher's exclusion list, the same pattern already used for `favicon.ico`. (2) `LoginForm.tsx` called `router.refresh()` immediately after `router.replace("/")` in the same tick; Next 16's App Router silently dropped the server's pending `NEXT_REDIRECT` to `/onboarding` as a result, leaving every real magic-link sign-in stuck on "Signing you in…" forever — confirmed via direct reproduction, then fixed by removing the redundant `refresh()` call (`router.replace()` alone already fetches the destination's RSC payload).
